Dataset Analysis

This dataset contains 6.744.091 rows of data ready to be used for analysis. It also contains 18 columns divided into categorical, numerical, datetime, text, and geolocation values that are distributed in the following way:

Datetime Columns:

  • fecha, unlock_date, lock_date: These columns track the date and time of bike usage. Useful for time-based analysis. These columns do not allow null values and the dates can take values between June 01 of 2021 to February 18 of 2023.

Numerical Columns:

  • idBike: Unique identifier for each bike, useful for aggregating trips by bike. It does not allow null values. These IDs range is from 0 to 8.090, but the number of unique bikes in operation in this dataset is 4.932.
  • trip_minutes: Duration of each trip, a critical metric to analyze ride times. It does not allow null or negative values, but a trip can last 0 minutes.
  • station_unlock, station_lock, dock_unlock, dock_lock: These represent the IDs of the docking stations and individual docks, allowing analysis of station usage. These columns can be null when the bike was not taken or left at a station. The station IDs range is from 1 to 889, and the dock IDs range is from 1 to 30.

Geolocation Columns:

  • geolocation_unlock, geolocation_lock: Contains a dictionary with location data for the unlock and lock events, considering the latitude and longitude of the geolocation point. It does not allow null values.

Text Columns:

  • address_unlock, address_lock: Provides the address where the bike trip started and ended. It allows null values.
  • unlock_station_name, lock_station_name: Contains the names of the stations for the unlock and lock events, useful for station popularity analysis. These columns allow null values when the bike was not taken or left at a station.

Categorical Columns:

  • fleet: Could be used to group or filter bikes based on their fleet. It does not allow null values and can take the values 1 or 2.
  • locktype, unlocktype: Can take the value "STATION" if the bike was taken or left at a station, or "FREE" if the bike was taken or left elsewhere. These columns do not allow null values.
