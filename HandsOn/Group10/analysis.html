Dataset Analysis

This dataset contains 6,744,091 rows of data ready to be used for analysis. It also contains 18 columns divided into categorical, numerical, datetime, text, and geolocation values that are distributed as follows:

1. Datetime Columns

  • fecha, unlock_date, lock_date: Track the date and time of bike usage. Useful for time-based analysis. These columns do not allow null values and range between June 1, 2021, to February 18, 2023.

2. Numerical Columns

  • idBike: Unique identifier for each bike. It does not allow null values. The range is from 0 to 8,090, with 4,932 unique bikes in operation.
  • trip_minutes: Duration of each trip. It does not allow null or negative values, though trips can last 0 minutes.
  • station_unlock, station_lock, dock_unlock, dock_lock: IDs of docking stations and docks. These columns can be null when a bike is not taken or left at a station. Station IDs range from 1 to 889, and dock IDs range from 1 to 30.

3. Geolocation Columns

  • geolocation_unlock, geolocation_lock: Contains location data for unlock and lock events, based on latitude and longitude. Null values are not allowed.

4. Text Columns

  • address_unlock, address_lock: Address where trips start and end. These columns allow null values.
  • unlock_station_name, lock_station_name: Names of the stations for unlock and lock events. These columns allow null values when bikes are not taken or left at a station.

5. Categorical Columns

  • fleet: Groups bikes based on their fleet. Null values are not allowed, and it can take values of 1 or 2.
  • locktype, unlocktype: Indicates whether the bike was taken or left at a station ("STATION") or elsewhere ("FREE"). Null values are not allowed.
