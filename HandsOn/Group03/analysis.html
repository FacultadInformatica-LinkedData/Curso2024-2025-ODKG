Dataset 1: Energy consumption in municipal buildings. Monthly data

  • YEAR - Numeric Value from 2020-2024
  • MONTH - Numeric
  • BUILDING USE - String, values: Centros administrativos, Centros culturales y bibliotecas, centros deportivos, centros escolares, centros mixtos, centros sociales, otras dependencias, salud, seguridad y emergencias
  • BUILDING - String
  • ADDRESS - String Format: "'STREET TYPE' 'STREET NAME' 'STREET NUMBER', 'POSTAL CODE' 'CITY'"
  • STREET TYPE - String
  • STREET NAME - String
  • STREET NUMBER - String
  • POSTAL CODE - Integer, optional
  • CITY - String
  • DISTRICT - String in capital letters
  • DISTRICT CODE - Number
  • NEIGHBORHOOD - String in capital letters
  • NEIGHBORHOOD CODE - Number
  • LATITUDE - Number
  • LONGITUDE - Number
  • SENSOR - String
  • SENSOR TYPE - String
  • ENERGY CLASS - String, values: Procesado, externo, telemedida
  • ENERGY UNITS - String, values: Activa Contador Gral -Bus, Activa Interruptor Gral, Activa Interruptor Gral - Bus, Gas, Ninguno, Reactiva tarifa 3.0A, Reactiva Tarifa 3.1A
  • CONSUMPTION - String, values: kvarh (Energía activa), kWh (Energía reactiva), m3 (Gas)

Dataset 2: Madrid housing prices and details

  • TITLE - String, title of the property
  • CITY - String, Madrid
  • NEIGHBORHOOD - String
  • DISTRICT - String
  • SQUARE_METERS - Number, from 10 to 1000
  • NUM_OF_ROOMS - Number, from 1 to 16
  • ADDRESS - String, optional
  • RENT_PRICE - Number, in euros, from 303 to 84 000
  • RENT_BY_ROOM_PRICE - Number, in euros, from 100 to 9000
  • RENT_BY_SQ_METER - Number in euros, from 1 to 210
  • ENERGY_CERTIFICATE - String, optional, values: A, B, C, D, E, F
  • uuid,uuid2,uuid3,uuid4 - String, python generated values, serve as ID

Resource Naming Strategy

NOTE: The domain for this ontology will be changed in the future to reflect its final hosting location.

Ontology IRI

The current IRI for the ontology is:

    http://www.semanticweb.org/ontology/SmartBarrioMadrid

A version number can be added if needed in the future.

Class IRIs

The IRI pattern for classes follows this structure:

    http://www.semanticweb.org/ontology/SmartBarrioMadrid#{Classname}

For example:

  • Class: http://www.semanticweb.org/ontology/SmartBarrioMadrid#District

Property IRIs

The IRI pattern for properties follows this structure:

    http://www.semanticweb.org/ontology/SmartBarrioMadrid#{Propertyname}

For example:

  • Property: http://www.semanticweb.org/ontology/SmartBarrioMadrid#population

Individual IRIs

The IRI pattern for individuals (instances) follows this structure:

    http://www.semanticweb.org/resource/{Classname}/{id}

For example:

  • Individual: http://www.semanticweb.org/resource/District/retiro

Once the domain is changed, the new IRI structure will follow the same resource naming pattern but under the new domain. This ensures that resources are easy to identify and the resource types are clear from their IRIs.
