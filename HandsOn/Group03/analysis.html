  • YEAR - Numeric Value from 2020-2024
  • MONTH - Numeric
  • BUILDING USE - String, values: Centros administrativos, Centros culturales y bibliotecas, centros deportivos, centros escolares, centros mixtos, centros sociales, otras dependencias, salud, seguridad y emergencias
  • BUILDING - String
  • ADDRESS - String Format: "'STREET TYPE' 'STREET NAME' 'STREET NUMBER', 'POSTAL CODE' 'CITY'"
  • STREET TYPE - String
  • STREET NAME - String
  • STREET NUMBER - String
  • POSTAL CODE - Integer
  • CITY - String

  • DISTRICT CODE - Number
  • NEIGHBORHOOD - String in capital letters
  • NEIGHBORHOOD CODE - Number
  • X-COORDINATE - Number
  • Y-COORDINATE - Number
  • LATITUDE - Number
  • LONGITUDE - Number
  • SENSOR - String
  • SENSOR TYPE - String
  • ENERGY CLASS - String, values: Procesado, externo, telemedida

  • NEIGHBORHOOD - String
  • DISTRICT CODE - Number, from 1 to 21

  • ADDRESS - String, optional
  • STREET NUM - Number, optional

Resource Naming Strategy

NOTE: The domain for this ontology will be changed in the future to reflect its final hosting location.

Ontology IRI

The current IRI for the ontology is:

    http://www.semanticweb.org/ontology/SmartDistrictMadrid

A version number can be added if needed in the future.

Class IRIs

The IRI pattern for classes follows this structure:

    http://www.semanticweb.org/ontology/SmartDistrictMadrid#{Classname}

For example:

  • Class: http://www.semanticweb.org/ontology/SmartDistrictMadrid#District

Property IRIs

The IRI pattern for properties follows this structure:

    http://www.semanticweb.org/ontology/SmartDistrictMadrid#{Propertyname}

For example:

  • Property: http://www.semanticweb.org/ontology/SmartDistrictMadrid#population

Individual IRIs

The IRI pattern for individuals (instances) follows this structure:

    http://www.semanticweb.org/resource/{Classname}/{id}

For example:

  • Individual: http://www.semanticweb.org/resource/District/retiro

Once the domain is changed, the new IRI structure will follow the same resource naming pattern but under the new domain. This ensures that resources are easy to identify and the resource types are clear from their IRIs.
