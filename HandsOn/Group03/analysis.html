Analysis of the datasets

Dataset 1: Universities and colleges

The dataset contains information about universities and colleges and other educational institutions in Madrid.

  • PK - primary key, government assigned number that identifies every facility, number
  • UNIVERSITY - String, name of the university if institute is a part of any, if not, then it's the same name like the school itself
  • SCHOOL - String, name of the school
  • TRANSPORT - String, what transport can be used to get there
  • ACCESSIBILITY - Number, from 0 to 3, discrete values
  • WEBSITE - String, page of the school on the www.madrid.es website
  • STREET NAME - String
  • STREET TYPE - String, AVENIDA, CALLE, CARRETERA, GLORIETA, PASEO, PLAZA, RONDA, TRAVESIA
  • NUM - Number, optional, number of the building on the street
  • FLOOR - Number, optional
  • ORIENTATION - String, optional, what bloque is it
  • CITY - String, Madrid
  • PROVINCE - String, Madrid
  • POSTAL CODE - Number, from 28 001 to 28 051
  • NEIGHBORHOOD CODE - Number, from 1 to 8
  • NEIGHBORHOOD - String
  • DISTRICT CODE - Number, from 1 to 21
  • DISTRICT - String
  • LATITUDE - Number
  • LONGITUDE - Number
  • TELEPHONE - String, optional
  • FAX - String, optional
  • EMAIL - String, optional

Dataset 2: Energy consumption in municipal buildings. Monthly data

  • YEAR - Numeric Value from 2020-2024
  • MONTH - Numeric
  • BUILDING USE - String, values: Centros administrativos, Centros culturales y bibliotecas, centros deportivos, centros escolares, centros mixtos, centros sociales, otras dependencias, salud, seguridad y emergencias
  • BUILDING - String
  • ADDRESS - String Format: "'STREET TYPE' 'STREET NAME' 'STREET NUMBER', 'POSTAL CODE' 'CITY'"
  • STREET TYPE - String
  • STREET NAME - String
  • STREET NUMBER - String
  • POSTAL CODE - Integer
  • CITY - String
  • DISTRICT - String in capital letters
  • DISTRICT CODE - Number
  • NEIGHBORHOOD - String in capital letters
  • NEIGHBORHOOD CODE - Number
  • X-COORDINATE - Number
  • Y-COORDINATE - Number
  • LATITUDE - Number
  • LONGITUDE - Number
  • SENSOR - String
  • SENSOR TYPE - String
  • ENERGY CLASS - String, values: Procesado, externo, telemedida
  • ENERGY GROUP - String, values: Active Energy, reactive Energy, Gas
  • ENERGY UNITS - String, values: Activa Contador Gral -Bus, Activa Interruptor Gral, Activa Interruptor Gral - Bus, Gas, Ninguno, Reactiva tarifa 3.0A, Reactiva Tarifa 3.1A
  • CONSUMPTION - String, values: kvarh (Energía activa), kWh (Energía reactiva), m3 (Gas)

Dataset 3: Madrid housing prices and details

  • ID - primary key, number, from 1 to 20113
  • TITLE - String, title of the property
  • CITY - String, Madrid
  • NEIGHBORHOOD - String
  • DISTRICT CODE - Number, from 1 to 21
  • DISTRICT - String
  • SQUARE METERS - Number, optional, from 10 to 1000
  • NUM OF ROOMS - Number, from 1 to 16
  • NUM OF BATHROOMS - Number, optional, from 1 to 15
  • STREET NAME - String, optional
  • STREET NUM - Number, optional
  • RENT PRICE - Number, in euros, from 303 to 84 000
  • RENT BY ROOM PRICE - Number, in euros, from 100 to 9000
  • BUY PRICE - Number, in euros, from 36 000 to 2 500 000
  • ENERGY CERTIFICATE - String, optional, values: A, B, C, D, E, F

Resource Naming Strategy

NOTE: The domain for this ontology will be changed in the future to reflect its final hosting location.

Ontology IRI

The current IRI for the ontology is:

    http://www.semanticweb.org/ontology/SmartBarrioMadrid

A version number can be added if needed in the future.

Class IRIs

The IRI pattern for classes follows this structure:

    http://www.semanticweb.org/ontology/SmartBarrioMadrid/{Classname}

For example:

  • Class: http://www.semanticweb.org/ontology/SmartBarrioMadrid/Student
  • Class: http://www.semanticweb.org/ontology/SmartBarrioMadrid/University

Individual IRIs

The IRI pattern for individuals (instances) follows this structure:

    http://www.semanticweb.org/resource/{Classname}/{id}

For example:

  • Individual: http://www.semanticweb.org/resource/Student/johnDoe
  • Individual: http://www.semanticweb.org/resource/University/madridUniversity

Once the domain is changed, the new IRI structure will follow the same resource naming pattern but under the new domain. This ensures that resources are easy to identify and the resource types are clear from their IRIs.
