Dublin Bikes Data Analysis

1. Dataset Overview

The dataset analysed comes from the Dublin Bikes system, which provides real-time information on bike availability and station status.

Variable Type Description
system_id object Identifier for the bike system
last_reported object Timestamp of the latest station report
station_id int64 Unique identifier for each station
num_bikes_available int64 Number of available bikes at each station
num_docks_available int64 Number of available docks at each station
is_installed bool Station installation status
is_renting bool Station renting status
is_returning bool Station returning status
name object Station name
short_name float64 Short name (missing data)
address object Station address
lat float64 Latitude
lon float64 Longitude
region_id float64 Region identifier (missing data)
capacity int64 Total capacity of each station

3. Resource Naming Strategy

The following resource naming strategy was applied to the dataset for consistency and clarity in Linked Data and RDF representations:

  • Station URIs: Each bike station is identified using the pattern: http://dublinbikesharing.org#Station_[station_id]
  • Bike Availability URIs: Each station's bike availability is captured using the pattern: http://dublinbikesharing.org#Station_[station_id]_BikesAvailable
  • Dock Availability URIs: Each station's dock availability is captured using the pattern: http://dublinbikesharing.org#Station_[station_id]_DocksAvailable

This strategy ensures that each resource is uniquely identified and can be referenced easily in a semantic web context.
