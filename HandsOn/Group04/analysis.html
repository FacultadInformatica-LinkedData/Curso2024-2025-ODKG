Dataset Analysis: Public Parking in Madrid

1. Data Analysis

This dataset includes information about public parking locations in Madrid, such as the name, schedule, accessibility, transportation options, and more. There are 66 instances with 30 variables each one.

Variables

  • PK (integer): Unique identifier for each parking lot, useful for internal reference or linking records across systems.
  • NOMBRE (string): Name of the parking lot, essential for user navigation and identification.
  • DESCRIPCION-ENTIDAD (string, may contain nulls): Entity description providing additional context about the parking facility.
  • HORARIO (string): Operational hours of the parking facility, crucial for accessibility and user planning.
  • EQUIPAMIENTO (string, may contain nulls): Equipment available at the parking lot, useful for understanding available services.
  • TRANSPORTE (string, may contain nulls): Nearby public transportation options, useful for linking with transportation systems.
  • DESCRIPCION (string): Description of parking spaces, including the number of spots for cars, motorcycles, bicycles, and electric vehicles.
  • ACCESIBILIDAD (integer): Accessibility information, potentially useful for users with disabilities, its value is 1 or 0 depending on if it is adaptable for user with disabilities.
  • CONTENT-URL (string): URL for further details on the parking facility, useful for external linking and further information.
  • NOMBRE-VIA (string): Street name where the parking facility is located, essential for geographical navigation.
  • CLASE-VIAL (string): Type of road (e.g., CALLE), providing urban classification of the street.
  • TIPO-NUM (string): Type of numbering (e.g., 'V' for varied), relevant for official street addresses.
  • NUM (integer): Street number, essential for locating the facility.
  • PLANTA (string, may contain nulls): Floor level, relevant for multi-level parking facilities.
  • PUERTA (string, may contain nulls): Door number, if applicable.
  • ESCALERAS (string, may contain nulls): Stairs information, relevant for accessibility.
  • ORIENTACION (string, may contain nulls): Orientation of the facility.
  • LOCALIDAD (string): City or town where the parking lot is located.
  • PROVINCIA (string): Province where the parking lot is located.
  • CODIGO-POSTAL (string): Postal code of the parking facility.
  • COD-BARRIO (integer): Code for the neighborhood, which can be used for geographical or urban analysis.
  • BARRIO (string): Neighborhood where the parking facility is located.
  • COD-DISTRITO (integer): Code for the district, useful for geographical referencing.
  • DISTRITO (string): District where the parking lot is located, relevant for administrative purposes.
  • COORDENADA-X (float): X-coordinate of the facility for GIS-based systems.
  • COORDENADA-Y (float): Y-coordinate of the facility for GIS-based systems.
  • LATITUD (float): Latitude for geographical location.
  • LONGITUD (float): Longitude for geographical location.
  • TELEFONO (string, may contain nulls): Contact phone number, if available.
  • FAX (string, may contain nulls): Fax number, if available.
  • EMAIL (string, may contain nulls): Email address for contact purposes.
  • TIPO (string): Indicates the type of facility, such as whether it is a public or private parking lot.

Data Summary

  • Number of parking spots per facility: Varies by facility, e.g., 344 for cars, 25 for motorcycles, 10 for bicycles at one location.
  • Parking types: Includes spaces for cars, electric cars, motorcycles, and bicycles.
  • District code (COD-DISTRITO): Range from 1 to 21.
  • Neighborhood code (COD-BARRIO): Range from 1 to 7.

Resource Naming Strategy

Base URI

We have defined that our base URI will be http://www.publicParkingMadrid.org/opendata/handsOn/group04/

URI for Classes and Properties

For classes and properties in our ontology, we use a hash (#) to indicate that they are part of the ontology schema:

Classes and Properties URI: http://www.publicParkingMadrid.org/opendata/handsOn/group04/ontology/PublicParkingOntology#

Example:

  • http://www.publicParkingMadrid.org/opendata/handsOn/group04/ontology/PublicParkingOntology#ParkingLot
  • http://www.publicParkingMadrid.org/opendata/handsOn/group04/ontology/PublicParkingOntology#hasOperator

For resources or instances in our dataset, we use a slash (/) to define them as unique real-world entities:

Instances URI: http://www.publicParkingMadrid.org/opendata/handsOn/group04/resource/

Example:

  • http://www.publicParkingMadrid.org/opendata/handsOn/group04/resource/ParkingLot123

Dataset Analysis: Electric Vehicle Charging Stations in Madrid

1. Data Analysis

This dataset includes information about charging points for electric vehicles such us the Location, the type of connector or the number of units. This dataset can be found in Dataset Link. There are 125 instance with 19 variables each one.

Variables

  • OID_ (integer): Unique identifier for each charging station, useful for internal reference or linking records across systems.
  • ID_Punto (string): Identifier code for each charging station, which could be used to link with other datasets about infrastructure.
  • DISTRITO (string): The district where the charging station is located, useful for administrative and geographical linking.
  • BARRIO (string): The neighborhood where the charging station is located, useful for neighborhood-specific urban analysis.
  • ESTADO (string): The operational status of the station, which could be linked with other real-time status systems.
  • UBICACIÓN (string): The street address of the charging station, essential for geographical and navigation-based applications.
  • EMPLAZAMIENTO (string): The type of placement (e.g., public or private), useful for understanding accessibility and categorization of stations.
  • INFO_ADICIONAL (string, may contain nulls): Additional information about the station that may provide further context for urban services.
  • Nº_EQUIPOS (integer): The number of charging units at the station, essential for understanding capacity and potential usage analysis.
  • CARACTERÍSTICAS_EQUIPO (string): Details about the equipment, including the power and type of connectors, useful for linking with electric vehicle compatibility data.
  • OPERADOR (string, may contain nulls): The operator of the charging station, which could be linked to service providers and market data.
  • GESTIÓN (string): Management details, such as whether the station is operated via a public or private agreement, useful for administrative purposes.
  • INFO_FECHA_APERTURA (string): Informational field indicating the planned opening date, which can be linked to project management systems.
  • FECHA_APERTURA (date): The actual opening date, useful for tracking operational history and linking with service reports.
  • FECHA_ALTA (date): The date the station was officially registered, which can be linked to administrative processes.
  • FECHA_EDICION (date): The date when the information was last updated, useful for ensuring data accuracy and linking with auditing systems.
  • COD_DIS (integer): Numeric code for the district, which can be linked with other datasets using standard district codes for geographical analysis.
  • COD_BAR (integer): Numeric code for the neighborhood, similar to COD_DIS, but for more localized urban analysis.
  • HORARIO (string, may contain nulls): Opening hours of the charging station, which can be linked to user accessibility data or real-time applications for finding available charging points.

Data Summary

  • Number of charging units per station (Nº_EQUIPOS): Range from 1 to 12.
  • District code (COD_DIS): Range from 1 to 21.
  • Neighborhood code (COD_BAR): Range from 1 to 21.
